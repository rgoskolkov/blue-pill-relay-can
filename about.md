Проект реализует интерфейс между выключателями света, релейного модуля в качестве исполняющего механизма для включения ламп, и Home Assistant по протоколу CanOpen, используя в качестве этого интерфейса плату blue pill на STM32F103C8T6.
Ключевые функции проекта
    1. Получать сигналы от физических выключателей света, и при изменнении состояния выключателя изменять состояние реле
    2. Получать сигналы от Home Assistant о включении выключении света
    3. Публиковать события для Home Assistant сообщая актуальное состояние каждого реле
    4. Переключать конкретное реле в релейном модуле по сигналу от выключателей и Home Assistant
Дополнительный функционал проекта
    1. С помощью встроенного на плате светодиода извещать о происходящих процессах

Используемые компоненты:
    Плата разработки blue pill на STM32F103C8T6
    Релейный модуль с опторазвязкой логики и реле, раздельным питанием логики и реле
    Транспортный модуль для CAN шины
    ST-Link v2 для upload кода на плату

Ипользуемые программные компоненты:
    Библиотеки (middleware):
        a. FreeRTOS для параллельного фунционирования различных задач (heap4)
        b. HAL для управления MCU
    ПО CubeMx для конфигурации платы и генерации кода
    ПО VSCode с плагином Platformio для разработки логики, сборки, заливки проекта на железо, и отладки
Структура кода:
    Все задачи, очереди, и прочее что связано с FreeRTOS задается в файле freertos.c

CAN сообщения:
COB-ID	Направление	Описание
0x180 + NodeID	TX — TPDO	Реле изменилось: [маска всех реле, индекс реле, состояние]
0x200 + NodeID	RX — RPDO	Команда на реле: [новая маска реле, 0, 0, ...]
0x580 + NodeID	TX — SDO Response	Ответ на конфиг запрос
0x600 + NodeID	RX — SDO Request	Запись Node ID или чтение состояния

  Как создать кнопки в Home Assistant для управления аддоном

  Вы можете легко создать их прямо в интерфейсе Home Assistant, используя "Помощники" (Helpers) и "Автоматизации" (Automations).

  1. Кнопка для авто-регистрации ("Provision Next Free")

   * Создаем кнопку:
       1. Зайдите в Настройки -> Устройства и службы -> вкладка Помощники.
       2. Нажмите + Создать помощника и выберите Кнопка.
       3. Назовите ее, например, CAN-шина: Найти новое устройство.
       4. Сохраните. У вас появится новая сущность input_button.can_shina_nayti_novoe_ustroystvo.

   * Создаем автоматизацию:
       1. Зайдите в Настройки -> Автоматизации и сцены.
       2. Нажмите + Создать автоматизацию -> Создать новую автоматизацию.
       3. Триггер (Когда): Тип Состояние, сущность — ваша новая кнопка (input_button.can_shina_nayti_novoe_ustroystvo).
       4. Действие (То): Тип Вызвать службу.
           * Служба: MQTT: Publish
           * Topic: can_helper/command
           * Payload: provision_next_free

  Теперь, когда вы нажмете эту кнопку на своей панели управления, Home Assistant автоматически отправит команду нашему аддону.

  2. Интерфейс для регистрации с конкретным ID

  Здесь нам понадобится два помощника и одна автоматизация.

   * Создаем помощников:
       1. Помощник 1 (Поле для ввода числа): Создайте помощника типа Число. Назовите его CAN-шина: ID для нового устройства. Задайте диапазон
          от 1 до 126.
       2. Помощник 2 (Кнопка): Создайте помощника типа Кнопка. Назовите CAN-шина: Назначить указанный ID.

   * Создаем автоматизацию:
       1. Триггер: Нажатие кнопки CAN-шина: Назначить указанный ID.
       2. Действия: Здесь будет два шага:
           * Шаг 1: Отправка ID
               * Служба: MQTT: Publish
               * Topic: can_helper/specific_id
               * Payload (нужно будет переключиться в режим YAML и вставить шаблон): {{
                 states('input_number.can_shina_id_dlya_novogo_ustroystva') | int }}
           * Шаг 2: Отправка команды
               * Служба: MQTT: Publish
               * Topic: can_helper/command
               * Payload: provision_specific

  Теперь вы можете добавить этих помощников на любую панель в Home Assistant и получите полноценный UI для управления регистрацией.
