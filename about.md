Проект реализует интерфейс между выключателями света, релейного модуля в качестве исполняющего механизма для включения ламп, и Home Assistant по протоколу CanOpen, используя в качестве этого интерфейса плату blue pill на STM32F103C8T6 (с возможным переиспользованием на dev board black pill v3.0 на STM32F4xx в будущем).
Ключевые функции проекта
    1. Получать сигналы от физических выключателей света, как кнопочного типа (нажал отпустил) так и классических, удерживающих режимы вкл выкл
    2. Получать сигналы от Home Assistant о включении выключении света
    3. Отвечать на запросы Home Assistant сообщая актуальное состояние каждого реле из свего релейного модуля
    4. Переключать конкретное реле в релейном модуле по сигналу от выключателей и Home Assistant
Дополнительный функционал проекта
    1. С помощью встроенного на плате светодиода извещать о происходящих процессах
    2. С помощью дополнительно припаянной к плате кнопки, конфигурировать slave id устройтва на шине modbus

Используемые компоненты:
    Плата разработки blue pill на STM32F103C8T6 (так же возможно использование STM32F103C6T6 и в будующем перейти на black pill STM32F4....)
    Релейный модуль с опторазвязкой логики и реле, раздельным питанием логики и реле
    Плата конвертера TTL в RS485 с автоматическим управлением направления передачи
    ST-Link v2 для upload кода на плату

Ипользуемые программные компоненты:
    Библиотеки (middleware):
        a. Библиотека для работы с протоколом Modbus RTU https://github.com/alejoseb/Modbus-STM32-HAL-FreeRTOS0.0
        b. FreeRTOS для параллельного фунционирования различных задач (heap4)
        c. HAL для управления MCU
    ПО CubeMx для конфигурации платы и генерации кода
    ПО VSCode с плагином Platformio для разработки логики, сборки, заливки проекта на железо, и отладки
Структура кода:
    Все задачи, очереди, и прочее что связано с FreeRTOS задается в файле freertos.c

CAN сообщения:
COB-ID	Направление	Описание
0x180 + NodeID	TX — TPDO	Реле изменилось: [маска всех реле, индекс реле, состояние]
0x200 + NodeID	RX — RPDO	Команда на реле: [новая маска реле, 0, 0, ...]
0x580 + NodeID	TX — SDO Response	Ответ на конфиг запрос
0x600 + NodeID	RX — SDO Request	Запись Node ID или чтение состояния
